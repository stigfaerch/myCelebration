# Plan 02-01: /admin/deltagere — CRUD

## Objective
Implementér fuldt fungerende deltagerstyring på `/admin/deltagere`. Gæster kan oprettes, redigeres, slettes. Admin kan kopiere UUID-URL, åbne SMS med udfyldt skabelon og acceptere invitation på vegne af gæst.

## Context
- `.planning/phases/02-admin-deltagere-information/02-CONTEXT.md`
- `.planning/phases/01-foundation/01-03-SUMMARY.md` — handoff-noter om admin route-struktur og server actions

## Codebase Conventions
- Admin-sider i `src/app/(admin)/admin/` (sidebar layout arves automatisk)
- Supabase: `import { supabaseServer } from '@/lib/supabase/server'`
- Server Actions: `'use server'` + `revalidatePath('/admin/deltagere')`
- shadcn/ui tilgængeligt: button, input, label, separator, sheet, dropdown-menu, sonner
- Brug `import Link from 'next/link'` — ikke `<a>` tags
- Tailwind v4 (CSS-based)

## Tasks

### Task 1: Guest server actions
**Action**: Installér først `lucide-react` (bruges i alle Phase 2 komponenter): `npm install lucide-react`

Opret `src/lib/actions/guests.ts` med følgende server actions:

```typescript
'use server'
import { revalidatePath } from 'next/cache'
import { supabaseServer } from '@/lib/supabase/server'

export type GuestType = 'main_person' | 'family' | 'friend' | 'screen'
export type TaskParticipation = 'none' | 'easy' | 'all'

export interface GuestFormData {
  name: string
  type: GuestType
  relation?: string
  age?: number | null
  gender?: string
  email?: string
  phone?: string
  task_participation: TaskParticipation
  default_page?: string        // only for type=screen
  is_primary_screen?: boolean  // only for type=screen
}

export async function getGuests() {
  const { data, error } = await supabaseServer
    .from('guests')
    .select('*')
    .order('name')
  if (error) throw new Error(error.message)
  return data
}

export async function getGuest(id: string) {
  const { data, error } = await supabaseServer
    .from('guests')
    .select('*')
    .eq('id', id)
    .single()
  if (error) throw new Error(error.message)
  return data
}

export async function createGuest(formData: GuestFormData) {
  const { error } = await supabaseServer.from('guests').insert(formData)
  if (error) throw new Error(error.message)
  revalidatePath('/admin/deltagere')
}

export async function updateGuest(id: string, formData: GuestFormData) {
  const { error } = await supabaseServer
    .from('guests')
    .update(formData)
    .eq('id', id)
  if (error) throw new Error(error.message)
  revalidatePath('/admin/deltagere')
}

export async function deleteGuest(id: string) {
  const { error } = await supabaseServer.from('guests').delete().eq('id', id)
  if (error) throw new Error(error.message)
  revalidatePath('/admin/deltagere')
}

export async function acceptInvitation(id: string) {
  const { error } = await supabaseServer
    .from('guests')
    .update({
      invitation_accepted: true,
      invitation_accepted_by: 'admin',
    })
    .eq('id', id)
  if (error) throw new Error(error.message)
  revalidatePath('/admin/deltagere')
}
```

Tilføj også hjælpefunktion til at generere gæste-URL (bruges i liste + SMS):
```typescript
export function getGuestUrl(uuid: string): string {
  const baseUrl = process.env.NEXT_PUBLIC_APP_URL ?? `http://localhost:3000`
  return `${baseUrl}/${uuid}`
}
```

Tilføj `NEXT_PUBLIC_APP_URL` til `.env.local` med `http://localhost:3000` (placeholder — sættes til rigtig URL i Vercel env vars).

**Verify**: `npx tsc --noEmit` — ingen fejl.

**Done**: Server actions oprettet og TypeScript kompilerer.

### Task 2: Deltagere liste-side
**Action**: Opret `src/app/(admin)/admin/deltagere/page.tsx`:

Siden er en Server Component der:
1. Henter alle gæster med `getGuests()`
2. Viser en tabel med kolonner: Navn | Type | Invitation | Opgave-deltagelse | Handlinger
3. Handlinger pr. række (brug dropdown-menu fra shadcn/ui):
   - **Kopier UUID-URL** (client-side `navigator.clipboard.writeText(guestUrl)`) — vis sonner toast
   - **Opret SMS** — link: `sms:?body=${encodeURIComponent(smsText)}` (henter skabelon fra `app_settings`)
   - **Acceptér invitation** (server action, kun hvis `!invitation_accepted`)
   - **Rediger** — link til `/admin/deltagere/{id}/rediger`
   - **Slet** — knap med confirm dialog

Øverst på siden: "Tilføj deltager"-knap → linker til `/admin/deltagere/ny`.

Type-visning: brug badges med farver (main_person=lilla, family=blå, friend=grøn, screen=grå).

Invitation-visning: grønt check hvis accepteret, gråt hvis ikke.

SMS-skabelon hentes fra `app_settings` tabel (singleton row) med:
```typescript
const { data: settings } = await supabaseServer.from('app_settings').select('sms_template').single()
const template = settings?.sms_template ?? 'Hej {navn}! Her er dit link: {url}'
```
Erstat `{navn}` og `{url}` med gæstens data.

Opret `src/components/admin/GuestActions.tsx` som Client Component der håndterer clipboard-kopiering og SMS-link (disse kræver browser APIs).

**Verify**: Siden renderes uden TypeScript-fejl.

**Done**: Deltagere liste-side viser alle gæster med handlinger.

### Task 3: Opret/rediger formular
**Action**: Opret `src/app/(admin)/admin/deltagere/ny/page.tsx` og `src/app/(admin)/admin/deltagere/[id]/rediger/page.tsx`.

Opret delt `src/components/admin/GuestForm.tsx` (Client Component) med alle felter:
- **Navn** (text, required)
- **Type** (select: main_person, family, friend, screen) — påvirker hvilke felter der vises
- **Relation** (text, optional)
- **Alder** (number, optional)
- **Køn** (select: mand/kvinde/andet/ikke angivet, optional)
- **E-mail** (email, optional)
- **Telefon** (tel, optional)
- **Opgave-deltagelse** (select: ingen/lette opgaver/alle opgaver)
- **Default side** (text, kun hvis type=screen) — f.eks. `/gallery`, `/program`
- **Primær skærm** (checkbox, kun hvis type=screen)

Formularen submitter til en Server Action via `action` prop.

For "ny": opret `src/lib/actions/guests.ts`-action `createGuestAction(formData: FormData)` der parser FormData og kalder `createGuest()`, derefter `redirect('/admin/deltagere')`.

For "rediger": opret tilsvarende `updateGuestAction(id: string, formData: FormData)`, derefter `redirect('/admin/deltagere')`.

Slet-funktionalitet: knap i rediger-siden der kalder `deleteGuest(id)` med `redirect('/admin/deltagere')`.

**Verify**: Ny gæst kan oprettes og gemmes. Rediger-side loader eksisterende data og opdaterer korrekt.

**Done**: Fuld CRUD for gæster fungerer.
