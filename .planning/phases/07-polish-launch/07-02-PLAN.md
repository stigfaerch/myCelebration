<task id="2" title="Smoke tests for critical flows (R01, R17, R20, R23)">
<action>
Create 4 test files. Each file sketches the intended flow + uses `test.describe.skip` guard when DB access is unavailable.

1. `tests/smoke/auth.spec.ts` — R01 + R02:
   ```ts
   import { test, expect } from '@playwright/test'

   const SAMPLE_UUID = process.env.TEST_GUEST_UUID
   const GUEST_PASSWORD = process.env.TEST_GUEST_PASSWORD

   test.describe('guest auth flow', () => {
     test.skip(!SAMPLE_UUID || !GUEST_PASSWORD, 'Requires TEST_GUEST_UUID + TEST_GUEST_PASSWORD env')

     test('redirects to /enter when no cookie set', async ({ page, context }) => {
       await context.clearCookies()
       await page.goto(`/${SAMPLE_UUID}`)
       await expect(page).toHaveURL(new RegExp(`/${SAMPLE_UUID}/enter$`))
     })

     test('landing on forside after correct password', async ({ page }) => {
       await page.goto(`/${SAMPLE_UUID}/enter`)
       await page.getByLabel(/kodeord/i).fill(GUEST_PASSWORD!)
       await page.getByRole('button', { name: /fortsæt|enter|log ind/i }).click()
       await expect(page).toHaveURL(`/${SAMPLE_UUID}`)
     })

     test('redirects home for invalid UUID', async ({ page }) => {
       await page.goto('/00000000-0000-4000-8000-000000000000')
       // Middleware redirects invalid UUID → /
       await expect(page).toHaveURL(/\/(enter)?$/)
     })
   })

   test('admin login rejects wrong password', async ({ page }) => {
     await page.goto('/admin/login')
     await page.getByLabel(/kodeord/i).fill('definitely-wrong-password')
     await page.getByRole('button', { name: /log ind|enter/i }).click()
     await expect(page.getByText(/forkert kodeord|wrong password/i)).toBeVisible()
   })
   ```

   - [ ] supabase/migrations/001_initial_schema.sql
   - [ ] supabase/migrations/002_rls.sql
   - [ ] supabase/migrations/003_program_items_swap_rpc.sql
   - [ ] supabase/migrations/004_swap_accept_rpc.sql
   - [ ] supabase/migrations/005_realtime_rls.sql — also adds tables to supabase_realtime publication

   ## 3. Storage Buckets (Supabase Dashboard → Storage)
   - [ ] Create bucket `invitations` (public: true)
   - [ ] Create bucket `maps` (public: true)
   - [ ] Create bucket `images` (public: true)

   ## 4. Environment Variables
   Local (.env.local) + Vercel (Project Settings → Environment Variables):
   - [ ] NEXT_PUBLIC_SUPABASE_URL
   - [ ] NEXT_PUBLIC_SUPABASE_ANON_KEY
   - [ ] SUPABASE_SERVICE_ROLE_KEY

   - [ ] `npm i -g vercel` (once)

   2. Rate limiting on mutating guest actions (swap, upload, memory CRUD)
   3. Signed URLs + private `images` bucket
   4. Audit log table + logAudit() helper
   5. Zod schemas at server-action boundaries
   6. Delete-flow TOCTOU (reorder storage delete before DB delete)
   7. middleware.ts → proxy.ts rename (Next.js 16 canonical)
   8. Generated typed Supabase client
   9. Stale override cleanup on admin delete
   10. Partial unique index on guests.is_primary_screen
