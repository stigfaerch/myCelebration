<tasks>
<task id="1" title="/admin/billeder — Photos management">
<action>
1. Create `src/lib/actions/photos.ts`:
   - `getPhotos(filters?)` — select photos with guest join: `*, guests(id, name, type)`. Order by taken_at desc. Optional filters:
     - `after?: string` (timestamptz) — filter taken_at >= after
     - `before?: string` (timestamptz) — filter taken_at <= before
     - `active?: boolean` — filter is_active
   - `togglePhotoActive(id, is_active)` — `assertAdmin()`, update is_active, revalidatePath
   - `deletePhoto(id)` — `assertAdmin()`, delete photo row, revalidatePath
   - Generic error messages on all mutations

2. Create `src/app/(admin)/admin/billeder/page.tsx`:
   - Server Component that fetches `getPhotos()` (no initial filters) and renders PhotoManager

3. Create `src/components/admin/PhotoManager.tsx`:
   - Client component with:
   - **Filter bar**: "efter" and "før" datetime-local inputs. Apply button re-fetches (via server action or form resubmit)
   - **Photo grid/list**: each photo shows:
     - Thumbnail (use `storage_url` directly as `<img>` src, small size via CSS)
     - Guest name (from join)
     - Taken at timestamp (formatted da-DK)
     - is_active badge (green "Aktiv" / gray "Deaktiveret")
   - **Actions per photo**:
     - Toggle active/deactive button (eye/eye-off icon)
     - Delete button with confirmation
     - "Vis på skærm" button — calls `showOnPrimaryScreen('photo', photo.id)`. Use `useTransition` for pending state
   - All icon-only buttons must have `aria-label`
   - Empty state: "Ingen billeder endnu. Billeder uploades af gæster."
</action>

<task id="2" title="/admin/minder — Memories management">
<action>
1. Create `src/lib/actions/memories.ts`:

   - `deleteMemory(id)` — `assertAdmin()`, delete memory row, revalidate
   - Generic error messages on all mutations

2. Create `src/app/(admin)/admin/minder/page.tsx`:
   - Server Component that fetches `getMemories()` and renders MemoryManager

3. Create `src/components/admin/MemoryManager.tsx`:

   - **Memory list**: each memory shows:
     - Title, type badge (funny/solemn/everyday/milestone with distinct colors)
     - Guest name (from join)
     - when_date if set
     - Description preview (truncated to 100 chars)
     - Image thumbnail if image_url exists (CSS-sized small)
   - **Actions per memory**:
     - Edit button → inline edit form for title, description, when_date (NOT type or guest_id — admin respects guest ownership)
     - Delete button with confirmation
     - "Vis på skærm" button — calls `showOnPrimaryScreen('memory', memory.id)`. Use `useTransition`
   - All icon-only buttons must have `aria-label`
   - Type labels (Danish): funny → "Sjov", solemn → "Højtidelig", everyday → "Hverdag", milestone → "Milepæl"
   - Empty state: "Ingen minder endnu. Minder oprettes af gæster."
