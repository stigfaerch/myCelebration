<task id="3" title="Screen override shared actions">
<action>
1. Create `src/lib/actions/screen.ts`:
   - Export `ScreenOverrideType = 'page' | 'photo' | 'memory' | 'gallery' | 'program'`

   - `getScreenGuests()` — select guests where type = 'screen', return id, name, is_primary_screen
     (Used by "Vis på skærm" to let admin choose which screen)

   - `showOnPrimaryScreen(overrideType, refId)`:
     - `assertAdmin()`
     - Find guest with `is_primary_screen = true` and `type = 'screen'`
     - If no primary screen found, throw 'No primary screen configured'
     - Upsert into screen_state: { guest_id, current_override: overrideType, override_ref_id: refId, updated_at: now() }
     - `revalidatePath` as needed

   - `setScreenOverride(screenGuestId, overrideType, refId)`:
     - `assertAdmin()`
     - Upsert into screen_state: { guest_id: screenGuestId, current_override: overrideType, override_ref_id: refId, updated_at: now() }

   - `clearScreenOverride(screenGuestId)`:
     - `assertAdmin()`
     - Update screen_state: set current_override = null, override_ref_id = null
     - Or delete the row — either works since screen falls back to default page

2. These actions will be imported by PageManager (this plan) and by PhotoManager/MemoryManager (Plan 04-02)
</action>
