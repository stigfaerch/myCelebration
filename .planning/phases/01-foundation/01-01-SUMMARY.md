## Decisions Made
- **create-next-app naming restriction**: The directory `myCelebration` contains uppercase letters, which npm rejects as a package name. Worked around by scaffolding into a temp subdirectory `mycelebration/`, then rsync-moving files to the project root while preserving `.git`, `.gitignore`, `README.md`, `notes/`, and `.planning/`. The `package.json` name field is `mycelebration` (lowercase) ‚Äî this is correct npm behavior and has no runtime impact.
- **Tailwind CSS v4**: create-next-app defaulted to Tailwind v4 (not v3). shadcn/ui v4.4.0 detected this and initialized correctly for v4. Downstream plans using Tailwind should be aware the config format is `postcss.config.mjs` (not `tailwind.config.ts`).
- **shadcn/ui style**: `--defaults` flag selects New York style, neutral base color, CSS variables ‚Äî matches the plan specification.
- **Next.js version**: 16.2.4 was installed (latest at time of execution). This version uses Turbopack by default for `next dev`.
- **Vercel CLI**: Not installed in this environment. Vercel linking and preview deployment are pending manual steps (see Issues).
