# Plan 01-02: Database Schema & Supabase Setup

## Objective
Opret Supabase-projekt og skriv komplet database-migrering der dækker **hele appens** datakrav — ikke kun Phase 1. RLS-policies implementeres så gæsters service-role-baserede adgang er korrekt afskærmet via Next.js middleware.

## Context
- `.planning/phases/01-foundation/01-CONTEXT.md` — fasekontekst og tabeloversigt
- `.planning/PROJECT.md` — krav R01–R23 med datamodel-implikationer
- `notes/PROJECT_SPECIFICATION.md` — komplet feature-spec med alle felter

## Tasks

### Task 1: Opret Supabase-projekt og konfigurér env vars
**Action**:
1. Opret nyt Supabase-projekt på supabase.com. Notér `SUPABASE_URL`, `SUPABASE_ANON_KEY`, `SUPABASE_SERVICE_ROLE_KEY`.
2. Opret `.env.local` med:
   ```
   NEXT_PUBLIC_SUPABASE_URL=...
   NEXT_PUBLIC_SUPABASE_ANON_KEY=...
   SUPABASE_SERVICE_ROLE_KEY=...
   ADMIN_PASSWORD=...         # admin-kodeord (valgfrit under dev)
   GUEST_PASSWORD=...         # fælles deltager-kodeord (valgfrit under dev)
   ```
3. Tilføj `.env.local` til `.gitignore` (verificer det allerede er der).
4. Kør `vercel env add` for alle 5 env vars til Preview og Production miljø.
5. Installér Supabase client: `npm install @supabase/supabase-js`.
6. Opret `src/lib/supabase/server.ts` og `src/lib/supabase/client.ts` med henholdsvis service-role og anon-key klienter.

**Verify**: `src/lib/supabase/server.ts` kompilerer. `.env.local` eksisterer men er ikke tracked af git.

**Done**: Supabase-projekt oprettet, env vars konfigureret lokalt og på Vercel, Supabase-klienter oprettet.

### Task 2: Skriv komplet database-migrering
**Action**: Opret `supabase/migrations/001_initial_schema.sql` med følgende tabeller:

```sql
-- Enums
create type guest_type as enum ('main_person', 'family', 'friend', 'screen');
create type invitation_accepted_by as enum ('guest', 'admin');
create type task_participation as enum ('none', 'easy', 'all');
create type performance_status as enum ('no', 'maybe', 'yes');
create type choice_type as enum ('binary', 'multichoice', 'text');
create type swap_status as enum ('pending', 'accepted', 'cancelled');
create type memory_type as enum ('funny', 'solemn', 'everyday', 'milestone');
create type gallery_display_type as enum ('single', 'quad', 'frames');
create type gallery_source as enum ('photos', 'memories', 'both');
create type screen_override_type as enum ('page', 'photo', 'memory', 'gallery', 'program');

-- Gæster / deltagere / screens
create table guests (
  id uuid primary key default gen_random_uuid(),
  name text not null,
  type guest_type not null default 'friend',
  default_page text,                          -- kun for type=screen
  is_primary_screen boolean default false,   -- kun for type=screen
  relation text,
  age integer,
  gender text,
  email text,
  phone text,
  invitation_accepted boolean default false,
  invitation_accepted_by invitation_accepted_by,
  task_participation task_participation default 'none',
  created_at timestamptz default now()
);

-- Festinformation (single row)
create table fest_info (
  id uuid primary key default gen_random_uuid(),
  description jsonb,                          -- TipTap JSON
  invitation_url text,                        -- Supabase Storage URL
  created_at timestamptz default now()
);

-- Begivenheder (gudstjeneste, fest, etc.)
create table events (
  id uuid primary key default gen_random_uuid(),
  name text not null,
  description text,
  start_time timestamptz,
  address text,
  google_maps_embed text,
  map_image_url text,
  map_image_description text,
  sort_order integer default 0,
  created_at timestamptz default now()
);

-- Lokationer pr. begivenhed (parkering, toilet, etc.)
create table event_locations (
  id uuid primary key default gen_random_uuid(),
  event_id uuid not null references events(id) on delete cascade,
  title text not null,
  description text,
  sort_order integer default 0
);

-- Konfigurerbare valg (admin definerer)
create table choice_definitions (
  id uuid primary key default gen_random_uuid(),
  title text not null,
  type choice_type not null,
  options jsonb,                              -- for multichoice: ["option1", "option2"]
  sort_order integer default 0,
  created_at timestamptz default now()
);

-- Deltager-svar på valg
create table guest_choices (
  id uuid primary key default gen_random_uuid(),
  guest_id uuid not null references guests(id) on delete cascade,
  choice_definition_id uuid not null references choice_definitions(id) on delete cascade,
  value text,                                 -- "true"/"false" for binary, option text for multichoice, free text for text
  unique(guest_id, choice_definition_id)
);

-- Indslag (oprettet af gæster)
create table performances (
  id uuid primary key default gen_random_uuid(),
  guest_id uuid not null references guests(id) on delete cascade,
  title text not null,
  elements text[],                            -- ['tale', 'sang', 'underholdning']
  duration_minutes integer,
  status performance_status default 'no',
  created_at timestamptz default now()
);

-- Programpunkter
create table program_items (
  id uuid primary key default gen_random_uuid(),
  parent_id uuid references program_items(id) on delete set null,
  performance_id uuid references performances(id) on delete set null,
  title text not null,
  scheduled_time text,                        -- "19:30" som tekst
  sort_order integer default 0,
  created_at timestamptz default now()
);

-- Opgaver
create table tasks (
  id uuid primary key default gen_random_uuid(),
  name text not null,
  scheduled_time timestamptz,
  description jsonb,                          -- TipTap JSON
  capacity integer default 1,
  contact_host_for_swap boolean default false,
  created_at timestamptz default now()
);

-- Opgavetildelinger
create table task_assignments (
  id uuid primary key default gen_random_uuid(),
  task_id uuid not null references tasks(id) on delete cascade,
  guest_id uuid not null references guests(id) on delete cascade,
  unique(task_id, guest_id)
);

-- Bytte-ønsker
create table swap_requests (
  id uuid primary key default gen_random_uuid(),
  requester_assignment_id uuid not null references task_assignments(id) on delete cascade,
  desired_task_ids uuid[],                    -- opgaver requester ønsker at bytte til
  status swap_status default 'pending',
  created_at timestamptz default now()
);

-- Minder (oprettet af gæster)
create table memories (
  id uuid primary key default gen_random_uuid(),
  guest_id uuid not null references guests(id) on delete cascade,
  title text not null,
  type memory_type not null,
  description text,
  when_date text,                             -- fritekst-dato ("sommeren 2019")
  image_url text,                             -- Supabase Storage URL
  created_at timestamptz default now()
);

-- Billeder (taget af gæster)
create table photos (
  id uuid primary key default gen_random_uuid(),
  guest_id uuid not null references guests(id) on delete cascade,
  storage_url text not null,
  taken_at timestamptz default now(),
  is_active boolean default true,
  created_at timestamptz default now()
);

-- Statiske sider (admin opretter)
create table pages (
  id uuid primary key default gen_random_uuid(),
  slug text not null unique,
  title text not null,
  content jsonb,                              -- TipTap JSON
  is_active boolean default false,
  visible_from timestamptz,
  visible_until timestamptz,
  sort_order integer default 0,
  created_at timestamptz default now()
);

-- Galleri-konfiguration (single row)
create table gallery_config (
  id uuid primary key default gen_random_uuid(),
  filter_after timestamptz,
  filter_before timestamptz,
  source gallery_source default 'both',
  interval_seconds integer default 8,
  display_type gallery_display_type default 'single',
  show_memory_text boolean default false
);

-- Screen-state (realtime overskrivning pr. screen)
create table screen_state (
  guest_id uuid primary key references guests(id) on delete cascade,
  current_override screen_override_type,
  override_ref_id uuid,                       -- ref til photo.id, memory.id, page.id
  updated_at timestamptz default now()
);

-- App-indstillinger (single row — SMS-skabelon m.v.)
create table app_settings (
  id uuid primary key default gen_random_uuid(),
  sms_template text default 'Hej {navn}! Her er dit link til konfirmationsfesten: {url}'
);

-- Seed: opret singleton-rækker
insert into fest_info (id) values (gen_random_uuid());
insert into gallery_config (id) values (gen_random_uuid());
insert into app_settings (id) values (gen_random_uuid());
```

Kør migreringen via Supabase Dashboard SQL Editor eller Supabase CLI (`supabase db push`).

**Verify**: Alle tabeller eksisterer i Supabase Dashboard → Table Editor.

**Done**: Komplet database-skema migreret til Supabase.

### Task 3: Implementér RLS-policies
**Action**: Opret `supabase/migrations/002_rls.sql`. Aktiver RLS på alle tabeller og opret policies:

```sql
-- Aktiver RLS på alle tabeller
alter table guests enable row level security;
alter table fest_info enable row level security;
alter table events enable row level security;
alter table event_locations enable row level security;
alter table choice_definitions enable row level security;
alter table guest_choices enable row level security;
alter table performances enable row level security;
alter table program_items enable row level security;
alter table tasks enable row level security;
alter table task_assignments enable row level security;
alter table swap_requests enable row level security;
alter table memories enable row level security;
alter table photos enable row level security;
alter table pages enable row level security;
alter table gallery_config enable row level security;
alter table screen_state enable row level security;
alter table app_settings enable row level security;

-- Service role bypasser automatisk RLS (bruges i server-side kode via SUPABASE_SERVICE_ROLE_KEY)
-- Anon key bruges KUN til Realtime subscriptions på screen_state

-- Public read på ikke-sensitive tabeller (bruges når gæst er autentificeret via middleware)
-- Alle faktiske queries kører server-side med service role key — RLS er backup-beskyttelse

-- Tillad ingen adgang via anon key undtagen Realtime på screen_state
create policy "screen_state_realtime_read" on screen_state
  for select using (true);  -- Realtime subscription bruger anon key til at lytte

-- Alle andre tabeller: ingen anon adgang (service role håndterer alt)
-- (Ingen yderligere policies nødvendige — default deny når RLS er aktiveret)
```

Dokumentér i `src/lib/supabase/README.md`:
- `server.ts`: service role key — brugt til alle server actions og API routes
- `client.ts`: anon key — brugt KUN til Realtime subscriptions (screen_state lytning)
- Auth-logik ligger i Next.js middleware, ikke i Supabase

**Verify**: Supabase Dashboard viser RLS aktiveret på alle tabeller. Anon key kan ikke læse `guests` direkte (test i Dashboard).

**Done**: RLS aktiveret på alle tabeller, policies dokumenteret, DB access pattern README skrevet.
