# Plan 01-03: Auth Middleware, i18n & Base Layouts

## Objective
Implementér Next.js middleware der beskytter gæste- og admin-routes via UUID-validering og cookie-baseret passwordcheck. Opsæt next-intl med dansk (komplet) og engelsk (skeleton). Opret base layouts for admin (desktop), gæst (mobil) og screen (fullscreen).

## Context
- `.planning/phases/01-foundation/01-CONTEXT.md` — auth-arkitektur og URL-slugs
- `.planning/phases/01-foundation/01-01-PLAN.md` (output) — mappestruktur
- `.planning/phases/01-foundation/01-02-PLAN.md` (output) — Supabase-klienter, `guests`-tabel

## Tasks

### Task 1: Auth middleware
**Action**: Opret `src/middleware.ts` der håndterer:

**Gæste-routes** (`/[uuid]` og alle undersider):
1. Udtræk `uuid` fra URL-path
2. Tjek `guest_password` cookie — hvis ikke sat eller ikke matcher `GUEST_PASSWORD` env var: redirect til `/{uuid}/enter` (login-side oprettet i dette plan)
3. Hvis password OK: verificer UUID eksisterer i `guests`-tabel via service role klient
4. Hvis UUID ikke eksisterer: redirect til `/` (rodsiden)
5. Sæt `x-guest-id` header på request så layouts kan bruge det

**Admin-routes** (`/admin` og alle undersider):
1. Tjek `admin_password` cookie — hvis ikke sat eller ikke matcher `ADMIN_PASSWORD` env var: redirect til `/admin/login`
2. Ingen database-lookup nødvendig for admin

**Offentlige routes** (pass-through uden middleware-tjek):
- `/` — rodside
- `/admin/login` — admin login-side
- `/[uuid]/enter` — gæste-password-side
- `/_next/*`, `/favicon.ico`, statiske assets

Opret også:
- `src/app/[uuid]/enter/page.tsx` — simpel formular: "Indtast kodeord" → sætter `guest_password` cookie → redirect til `/{uuid}`
- `src/app/admin/login/page.tsx` — simpel formular: "Indtast kodeord" → sætter `admin_password` cookie → redirect til `/admin`
- `src/app/page.tsx` — rodside med tekst: "Har du fået en invitation? Kontakt værten for at få dit personlige link."

Cookies: `httpOnly: true, sameSite: 'lax', path: '/'`. Ingen expiry (session cookie) — kan justeres senere.

**Verify**:
- Besøg `http://localhost:3000/test-uuid` uden cookie → redirect til `/test-uuid/enter`
- Besøg `http://localhost:3000/admin` uden cookie → redirect til `/admin/login`
- Ugyldigt UUID efter password-check → redirect til `/`

**Done**: Auth middleware fungerer korrekt for alle tre scenarier.

### Task 2: next-intl opsætning
**Action**:
1. Installér `next-intl`
2. Opret `src/i18n/routing.ts` med `locales: ['da', 'en']` og `defaultLocale: 'da'`
3. Opret `messages/da.json` — komplet med alle statiske UI-tekster der kendes på nuværende tidspunkt (login-sider, rodsiden, generiske labels). Strukturér i namespaces: `common`, `auth`, `admin`, `guest`
4. Opret `messages/en.json` — skeleton med samme keys som `da.json`, værdier enten maskinoversat eller `"[EN] {dansk tekst}"` som placeholder
5. Konfigurér next-intl i `next.config.ts` og opret `src/i18n/request.ts`
6. Wrap root layout med `NextIntlClientProvider`

URL-strategi: **ingen locale-prefix i URL** (next-intl `pathnames`-strategi med implicit locale). Dansk er default og eneste aktive sprog i v1 — engelsk er skeleton til fremtidig brug.

**Verify**: `useTranslations('common')` virker i en Server Component uden fejl. `messages/da.json` og `messages/en.json` eksisterer.

**Done**: next-intl konfigureret, dansk komplet, engelsk skeleton på plads.

### Task 3: Base layouts
**Action**: Opret de tre base layouts der bruges i resten af projektet:

**Admin layout** (`src/app/(admin)/layout.tsx`):
- Desktop-first (`min-w-[1024px]`)
- Sidebar til venstre (placeholder nav-links for alle admin-sider)
- Main content area til højre
- Ingen bottom-nav

**Guest layout** (`src/app/[uuid]/layout.tsx`):
- Mobile-first (`max-w-[430px] mx-auto`)
- Læser `x-guest-id` header fra middleware
- Tjekker `guests.type` — hvis `screen`: render `ScreenLayout`, ellers: render `GuestLayout`
- `GuestLayout`: header + main content + placeholder bottom-nav (4 knapper: Hjem, Program, Kamera, Menu)
- `ScreenLayout`: fullscreen horisontal (`w-screen h-screen overflow-hidden`), ingen nav

Opret placeholder index-sider der blot viser layoutet:
- `src/app/(admin)/admin/page.tsx` — "Admin dashboard (coming soon)"
- `src/app/[uuid]/page.tsx` — renderer ScreenLayout eller GuestLayout baseret på gæst-type, viser "Velkommen {name}"

**Verify**:
- `/admin` med admin-cookie viser sidebar layout
- `/{valid-uuid}` med guest-cookie viser mobile guest layout
- Screen-type gæst viser fullscreen layout

**Done**: Alle tre layouts eksisterer og renders korrekt baseret på gæst-type.
