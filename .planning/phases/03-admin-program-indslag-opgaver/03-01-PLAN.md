# Plan 03-01: /admin/indslag + /admin/opgaver

## Objective
Implementér `/admin/indslag` som read-mostly oversigt (gæster opretter indslag, admin redigerer kun varighed og status) og `/admin/opgaver` med fuld CRUD inkl. tildeling af gæster og "kontakt værten"-flag der blokerer bytte-flow.

### Task 1: Performances server actions + /admin/indslag side

**Action**: Opret `src/lib/actions/performances.ts`:

```typescript
'use server'
import { revalidatePath } from 'next/cache'
import { supabaseServer } from '@/lib/supabase/server'

export type PerformanceType = 'speech' | 'toast' | 'music' | 'dance' | 'poem' | 'other'
export type PerformanceStatus = 'pending' | 'approved' | 'rejected' | 'scheduled'

export interface Performance {
  id: string
  guest_id: string
  type: PerformanceType
  title: string
  description: string | null
  duration_minutes: number | null
  sort_order: number
  status: PerformanceStatus
  created_at: string
}

export async function getPerformances() {
  const { data, error } = await supabaseServer
    .from('performances')
    .select('*, guests(name, type)')
    .order('created_at', { ascending: false })
  if (error) throw new Error(error.message)
  return data
}

export async function updatePerformanceDuration(id: string, duration_minutes: number | null) {
  const { error } = await supabaseServer
    .from('performances')
    .update({ duration_minutes })
    .eq('id', id)
  if (error) throw new Error(error.message)
  revalidatePath('/admin/indslag')
  revalidatePath('/admin/program')
}

export async function updatePerformanceStatus(id: string, status: PerformanceStatus) {
  const { error } = await supabaseServer
    .from('performances')
    .update({ status })
    .eq('id', id)
  if (error) throw new Error(error.message)
  revalidatePath('/admin/indslag')
}
```

Opret `src/app/(admin)/admin/indslag/page.tsx` (Server Component):
- Henter alle performances med `getPerformances()` (inkl. gæste-navn via join)
- Øverst: type-filter med knapper: Alle | Tale | Skål | Musik | Dans | Digt | Andet
  - Filter er client-side state (Client Component `src/components/admin/PerformanceList.tsx`)
- Liste per indslag: gæstenavn, type (badge), titel, beskrivelse (truncated), varighed, status (badge)
- Varighed: inline editerbart tal-input (blur/enter gemmer via `updatePerformanceDuration`)
- Status: dropdown med muligheder (pending/approved/rejected/scheduled) — kalder `updatePerformanceStatus`
- Indslag kan IKKE oprettes/slettes af admin (gæster ejer dem)

Type-labels på dansk:
- speech → Tale, toast → Skål, music → Musik, dance → Dans, poem → Digt, other → Andet

Status-labels:
- pending → Afventer, approved → Godkendt, rejected → Afvist, scheduled → Planlagt

Opret `src/components/admin/PerformanceList.tsx` (Client Component) med:
- Type-filter state
- Filtreret liste fra `initialPerformances` prop
- Inline varighed-redigering (number input, onBlur kalder server action)
- Status-dropdown (select element, onChange kalder server action)

**Verify**: `npx tsc --noEmit` — ingen fejl.

**Done**: `/admin/indslag` viser alle gæste-indslag med type-filter og admin kan redigere varighed og status.

### Task 2: Tasks server actions + /admin/opgaver CRUD

**Action**: Opret `src/lib/actions/tasks.ts`:

```typescript
'use server'
import { revalidatePath } from 'next/cache'
import { supabaseServer } from '@/lib/supabase/server'

export interface Task {
  id: string
  title: string
  description: string | null
  location: string | null
  due_time: string | null
  max_persons: number | null
  contact_host: boolean
  sort_order: number
  created_at: string
}

export interface TaskAssignment {
  id: string
  task_id: string
  guest_id: string
  is_owner: boolean
}

export async function getTasks() {
  const { data, error } = await supabaseServer
    .from('tasks')
    .select('*, task_assignments(*, guests(name, type))')
    .order('sort_order')
  if (error) throw new Error(error.message)
  return data
}

export async function createTask(formData: {
  title: string
  description?: string
  location?: string
  due_time?: string
  max_persons?: number | null
  contact_host: boolean
}) {
  const { error } = await supabaseServer.from('tasks').insert(formData)
  if (error) throw new Error(error.message)
  revalidatePath('/admin/opgaver')
}

export async function updateTask(id: string, formData: {
  title: string
  description?: string
  location?: string
  due_time?: string
  max_persons?: number | null
  contact_host: boolean
}) {
  const { error } = await supabaseServer.from('tasks').update(formData).eq('id', id)
  if (error) throw new Error(error.message)
  revalidatePath('/admin/opgaver')
}

export async function deleteTask(id: string) {
  const { error } = await supabaseServer.from('tasks').delete().eq('id', id)
  if (error) throw new Error(error.message)
  revalidatePath('/admin/opgaver')
}

export async function toggleContactHost(id: string, contact_host: boolean) {
  const { error } = await supabaseServer
    .from('tasks')
    .update({ contact_host })
    .eq('id', id)
  if (error) throw new Error(error.message)
  revalidatePath('/admin/opgaver')
}
```

Opret `src/components/admin/TaskForm.tsx` (Client Component):
- Felter: Titel (required), Beskrivelse (textarea), Sted, Tidspunkt (datetime-local), Maks personer (number), "Kontakt værten" (checkbox)
- Props: `task?: Task` (til redigering), `onSave: () => void`
- Submit kalder `createTask` eller `updateTask` afhængig af om `task` prop er givet

Opret `src/app/(admin)/admin/opgaver/page.tsx` (Server Component):

- Øverst: "Ny opgave"-knap åbner Sheet med `TaskForm`

- Per opgave: rediger-knap (Sheet med TaskForm pre-udfyldt), slet-knap (confirm)
- Vis tildelinger under hver opgave (gæste-navne)

**Verify**: `npx tsc --noEmit` — ingen fejl.

### Task 3: Opgave-tildelingskomponent + "kontakt værten" + flyt opgave

**Action**: Tilføj til `src/lib/actions/tasks.ts`:

```typescript
export async function assignGuestToTask(taskId: string, guestId: string) {
  // Check if assignment already exists
  const { data: existing } = await supabaseServer
    .from('task_assignments')
    .select('id')
    .eq('task_id', taskId)
    .eq('guest_id', guestId)
    .single()
  if (existing) return // Already assigned

  const { error } = await supabaseServer
    .from('task_assignments')
    .insert({ task_id: taskId, guest_id: guestId, is_owner: false })
  if (error) throw new Error(error.message)
  revalidatePath('/admin/opgaver')
}

export async function removeGuestFromTask(assignmentId: string) {
  const { error } = await supabaseServer
    .from('task_assignments')
    .delete()
    .eq('id', assignmentId)
  if (error) throw new Error(error.message)
  revalidatePath('/admin/opgaver')
}

export async function moveGuestToTask(assignmentId: string, newTaskId: string) {
  const { error } = await supabaseServer
    .from('task_assignments')
    .update({ task_id: newTaskId })
    .eq('id', assignmentId)
  if (error) throw new Error(error.message)
  revalidatePath('/admin/opgaver')
}
```

Opret `src/components/admin/TaskAssignmentManager.tsx` (Client Component):
- Props: `task` (med nested `task_assignments` + `guests`), `allGuests: Guest[]`, `allTasks: Task[]`
- Vis liste over tildelinger med fjern-knap (× knap kalder `removeGuestFromTask`)
- "Tilføj deltager"-select (dropdown med gæster der ikke er tildelt denne opgave endnu) + tilføj-knap
- "Flyt til anden opgave"-dropdown pr. tildeling (vælg opgave → kalder `moveGuestToTask`)
- "Kontakt værten"-toggle switch: viser advarsels-badge hvis aktivt; kalder `toggleContactHost`

Integrer `TaskAssignmentManager` i `/admin/opgaver/page.tsx` under hver opgave (expand/collapse).

Importér `getGuests` fra `@/lib/actions/guests` for at hente tilgængelige gæster (filtrér `type !== 'screen'`).

**Verify**: `npx tsc --noEmit` — ingen fejl.
